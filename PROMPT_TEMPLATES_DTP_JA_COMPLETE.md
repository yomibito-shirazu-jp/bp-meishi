# DTP業務向け AIプロンプト清書版（省略なし・実務投入用）

このファイルの各プロンプトは、**そのままコピーしてAIに送信できる完全版**です。
必要な差し替えは「原稿本文」「比較対象テキスト」「画像添付」などの業務データのみです。

---

## 1) 文章作成・リライト（一般読者向け）

```text
あなたはビジネス系Webメディアの編集長です。
以下の原稿を、専門知識のない20代〜30代のビジネスパーソンが3分で理解できる記事にリライトしてください。

【リライト要件】
- 文体は「です・ます」調で統一する。
- 文字数は420文字以上480文字以下にする。
- 見出しを必ず2つ入れる。見出しはMarkdown形式で「## 」から始める。
- 専門用語は、本文中で必ず平易な言葉に言い換えるか、直後に括弧で説明する。
- 抽象表現を減らし、具体的な効果を1つ以上明記する。
- 読者が行動に移しやすい締めの一文で終える。

【出力形式】
- 見出し付き本文のみを出力する。
- 前置き、注釈、自己評価、箇条書きの解説は出力しない。

【元原稿】
当社が新たに導入した人事評価システムは、従業員のコンピテンシーを定量的に測定することを目的としている。これまでのMBO（目標管理制度）に依存した評価体系を刷新し、360度評価を組み合わせることで、より多角的な視点からのアセスメントが可能となる。これにより、次世代リーダーの早期発掘およびタレントマネジメントの最適化を図る所存である。特に若手層のエンゲージメント向上において、この新たな評価基準は重要なファクターとして機能するだろう。
```

## 2) OCR・文字起こし整文（ケバ取り＋誤認識修正）

```text
あなたは出版社の実務ライターです。
以下の文字起こしテキストを、話者の意図を変えずに、読みやすい原稿へ整文してください。

【整文ルール】
1. フィラー（えーと、あのー、まあ、そのー等）は完全に削除する。
2. 不要な相槌（はい、なるほど、そうですね等）は削除する。
3. 文のねじれ、倒置、主語述語不一致を自然な語順へ修正する。
4. OCR誤認識や誤変換と思われる単語は、文脈に沿って正しい語へ修正する。
5. 句読点を適切に補い、意味の区切りで段落を分ける。
6. 数字・単位・固有名詞は可能な限り正規化する（例：20%は半角、製品名の表記統一）。

【禁止事項】
- 発言内容の要約・削除・追加解釈をしない。
- 事実関係を推測で改変しない。

【出力形式】
- 整文後の本文のみを出力する。
- 変更理由の説明は出力しない。

【文字起こしデータ】
えーと、今回のあの新製品の開発なんですけど、まあ、一番苦労したのはですね、やっぱりあの、コストダウンのところですかね。はい。設計チームとも何度も、えー、やり取りをして、部品の調達先もですね、見直したんですよ。なので、結果的には、当初の予定よりもですね、えー、20%くらい？コストを下げることに成功しまして。まあ、ほんとに皆の協力があったからこそ、できたのかなと、はい、そう思ってます。
```

## 3) 校閲・校正（指摘のみ出力）

```text
あなたは出版社の校閲部員です。
以下の原稿を厳密に校正し、修正候補を指摘してください。
原稿本文は書き換えず、指摘事項のみを出力してください。

【校正基準】
- 誤字脱字、助詞誤用、文法誤りを検出する。
- 表記ゆれを検出し、1つの推奨表記へ統一案を示す。
- 重複表現、不自然敬語、二重敬語を検出する。
- ら抜き・い抜き・誤用敬語を検出する。
- 送り仮名や漢字表記は一般的な商用文書基準に合わせる。

【出力形式（この形を厳守）】
- 【対象箇所】原文から該当部分をそのまま抜粋
  【修正案】修正後の表現
  【理由】誤りの種類と修正根拠

【原稿】
本日はお忙しい中、当社のWebセミナーにご参加いただきありがとうごさいます。これから、新しいクラウドサーバの導入手順についてご説明を行ないます。あらかじめ予定していたスケジュールよりも、少し早めに終了する見込みです。また、データのバックアップを行う際は、必ず専用のサーバーをご利用してください。後ほどアンケートを送らせて頂きますので、ご協力をお願い致します。分からない事があれば、いつでも質問を見れるようにチャット欄に書き込んでください。
```

- 言い換え
- 数値変更
- 日付変更
- 固有名詞変更

【出力形式（厳守）】
・変更箇所[1]
【初校】...
【再校】...
【変更内容】追加 / 削除 / 修正 の別を明記し、何がどう変わったかを1文で記載

## 5) PDF加工・編集（Pythonスクリプト生成）

```text
あなたはPython 3.10の業務自動化エンジニアです。
以下の要件を満たす、実行可能な完全なPythonコードを1ファイルで出力してください。

【実行環境】
- OS: Windows 11
- Python: 3.10
- ライブラリ: PyPDF2 最新版

【要件】
1. スクリプトと同じ階層の input フォルダ内にあるPDFをすべて取得する。
2. PDFをファイル名昇順で1つに結合する。
3. 各ページ右上に赤い「【社外秘】」透かしを付ける。透かしは同階層の watermark.pdf を重ねて実現する。
4. 結果を output.pdf として保存する。
5. 入力PDFが0件、watermark.pdfがない、読み取り失敗などの例外を日本語メッセージで処理する。

【出力条件】
- 最初から最後まで完全なPythonコードのみを出力する。
- コードの各処理ブロックに日本語コメントを付ける。
- requirements.txt相当として必要ライブラリ名をコード先頭コメントで明記する。
```

## 6) 組版指示書（DTPタグ付け）

```text
あなたは商業印刷案件を担当するDTPオペレーターです。
以下の原稿を、InDesign流し込み用に1行ずつ判定し、行頭へタグを付けてください。

【タグ定義】
- <H1> 章タイトル（最上位見出し）
- <H2> 節タイトル（中見出し）
- <Body> 通常本文
- <List> 箇条書き項目
- <Cap> 図表キャプション、注記

【判定ルール】
- 空行は出力しない。
- 句点のない短文でも見出し文脈なら <H2> を優先する。
- 「図」「表」で始まる行は原則 <Cap> とする。
- 「・」「-」「1.」等で始まる行は <List> とする。

【出力形式】
<Tag>本文
<Tag>本文
...

【原稿】
第一章 AI活用の基礎
AIを業務に導入するためのステップ
近年、AI技術の発展により業務効率化が急速に進んでいます。本章では、その基礎を解説します。
導入のメリット
AIを導入する主なメリットは以下の通りです。
・作業時間の短縮
・ヒューマンエラーの削減
・コストの最適化
図1：AI導入によるコスト削減の推移
まずは小さな業務からテスト導入することが成功の鍵となります。
```

## 7) レイアウト検出（画像/PDFスクリーンショット入力）

```text
あなたは雑誌DTPの構成検証担当です。
添付画像のレイアウトを解析し、以下を実施してください。

【タスク1: 要素抽出】
画像から読み取れるテキストを以下に分類して列挙する。
- 大タイトル
- 中見出し
- 本文
- 図表キャプション
- 図表番号

【タスク2: 図表番号検証】
- 画像内の図表番号（図1、図2、表1…）を抽出する。
- 本文中の参照表現（例: 図2参照）を抽出する。
- 連番抜け、重複、参照先不一致を検証し、エラー有無を明示する。

【タスク3: 構造化出力】
誌面構造をMarkdownで再構成する。

【出力形式】
1. 要素抽出結果
2. 図表番号検証結果
3. Markdown構造化結果
```

## 8) 原稿検証（図表参照整合性チェック強化版）

```text
あなたは技術書制作の品質管理担当です。
添付誌面画像に対して、図表番号と本文参照の整合性を厳密に検証してください。

【検証観点】
- 連番が 1,2,3... の順に続いているか
- 同一番号の重複使用がないか
- 本文で参照している図表が誌面に実在するか
- 誌面に存在する図表で本文未参照のものがないか

【出力形式】
- 検証サマリー（OK / NG）
- 問題一覧（番号、問題種別、詳細）
- 修正提案（実務での修正手順）
```

## 9) InDesign連携（ExtendScript .jsx 生成）

```text
あなたはAdobe InDesign 2024向けExtendScript開発者です。
以下要件を満たす .jsx の完全コードを作成してください。

【機能要件】
1. アクティブドキュメントの有無を確認し、未オープン時は警告ダイアログを表示して終了。
2. ドキュメント全体を検索対象にする。
3. 正規表現で「【...】」に一致する文字列を全件抽出する。
4. 抽出した文字列（括弧含む）へ、既存文字スタイル「強調文字」を適用する。
5. 完了時に「N箇所のテキストに文字スタイルを適用しました」と表示する。

【品質要件】
- try-catchで例外処理を実装する。
- 検索設定の初期化処理を入れる。
- コード全体に日本語コメントを付与する。
- そのまま test.jsx として保存・実行できる完全コードを出力する。

【出力形式】
- コードブロック1つで、完全な.jsxコードのみを出力する。
```

## 10) PDF赤字修正の反映（元テキストへの適用）

```text
あなたは校正反映担当の編集者です。
以下の修正指示を元テキストへ一字一句正確に反映し、修正後テキスト全文を出力してください。

【厳守事項】
- 指示の漏れをゼロにする。
- 指示されていない箇所は変更しない。
- 出力は修正後テキスト全文のみ。

【修正指示】
- 1段落目「最新のテクノロジー」を「最新のIT技術」に変更。
- 1段落目末尾に「詳細はWebサイトをご覧ください。」を追加。
- 2段落目の「非常に」を削除。
- 2段落目「スマートフォン」を「モバイル端末」に変更。

【元テキスト】
現代のビジネスにおいて、最新のテクノロジーを活用することは不可欠となっています。業務効率化を図るために、各社が様々なツールを導入しています。
現在、非常に多くのユーザーがスマートフォン経由でサービスにアクセスしています。そのため、UIの改善が急務となっています。
```
